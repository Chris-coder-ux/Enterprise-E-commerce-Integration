¿Qué son los Endpoints?

Concepto

Los endpoints son puntos de acceso específicos de la API de Verial que permiten realizar operaciones sobre datos (clientes, pedidos, productos, etc.).

En esta herramienta, puedes probar operaciones POST, que son aquellas que envían datos al servidor Verial para crear o actualizar información.

1

Conexión

El sistema se conecta a la API de Verial usando las credenciales configuradas.

2

Envío

Envía los datos JSON al endpoint seleccionado mediante una petición POST.

3

Procesamiento

Verial procesa la petición y ejecuta la operación solicitada.

4

Respuesta

El sistema muestra los resultados en formato tabla legible.

Funcionalidades

Selector de Endpoints

Selecciona el endpoint que deseas probar de una lista predefinida:

  • Crear Pedido: Envía un nuevo pedido a Verial
  • Actualizar Stock: Modifica el stock de productos
  • Registrar Cliente: Crea o actualiza clientes
  • Actualizar Precios: Actualiza precios de productos
  • Crear Artículo: Da de alta un nuevo producto

Editor de Parámetros JSON

Editor de texto multilínea para escribir los parámetros en formato JSON:

  • Valida el formato JSON antes de enviar
  • Resalta errores de sintaxis
  • Sugiere estructura básica según el endpoint

Botón de Prueba

Envía la petición POST al endpoint seleccionado:

  • Muestra indicador de carga durante el proceso
  • Gestiona errores de forma visual
  • Confirma éxito de la operación

Visualización de Resultados

Muestra los resultados de la petición en formato tabla:

  • Campo y valor por fila
  • Formato JSON en campos anidados
  • Copiar resultados al portapapeles

Cómo Usar las Pruebas

Pasos para Probar un Endpoint

Paso 1: Seleccionar Endpoint

En el menú desplegable "Endpoint", selecciona la operación que deseas probar:

Ejemplo: Crear Pedido (crear_pedido)

Paso 2: Introducir Parámetros JSON

En el campo de texto "Parámetros JSON", escribe la estructura de datos en formato JSON:

{
  "sesionwcf": 18,
  "Tipo": 5,
  "ID_Cliente": 1,
  "Contenido": [
    {
      "ID_Articulo": 123,
      "Uds": 2,
      "Precio": 29.99
    }
  ]
}

Paso 3: Hacer Clic en "Probar API"

Haz clic en el botón "Probar API" para enviar la petición:

  • Verás un indicador de carga mientras se procesa
  • El sistema se conectará con Verial
  • Espera a que aparezcan los resultados

Paso 4: Revisar Resultados

Los resultados aparecerán en una tabla debajo del formulario:

  • Cadena "Campo" muestra el nombre del campo
  • Cadena "Valor" muestra el valor devuelto
  • Los valores JSON se formatean automáticamente

Endpoints Disponibles

POST NuevoClienteWS

Función: Da de alta un nuevo cliente o modifica uno existente en Verial.

Parámetros Principales:
  • sesionwcf - Número de sesión (Requerido)
  • Id - ID del cliente para modificar (Opcional)
  • Tipo - Tipo de cliente: 1=Particular, 2=Empresa (Requerido)
  • NIF - NIF/CIF del cliente (Requerido)
  • Nombre - Nombre del cliente (Requerido)
  • Direccion - Dirección del cliente (Requerido)
  • Email - Correo electrónico (Opcional)

POST NuevoDocClienteWS

Función: Da de alta o modifica un documento de cliente (pedido, factura, albarán).

Parámetros Principales:
  • sesionwcf - Número de sesión (Requerido)
  • Id - ID del documento para modificar (Opcional)
  • Tipo - Tipo de documento: 1=Factura, 5=Pedido (Requerido)
  • ID_Cliente - ID del cliente (Requerido)
  • Contenido - Array con las líneas del documento (Requerido)
  • Pagos - Array con los pagos del documento (Opcional)

POST NuevaDireccionEnvioWS

Función: Da de alta una nueva dirección de envío en un cliente existente.

Parámetros Principales:
  • sesionwcf - Número de sesión (Requerido)
  • ID_Cliente - ID del cliente (Requerido)
  • Nombre - Nombre del destinatario (Requerido)
  • Direccion - Dirección de envío (Requerido)
  • CPostal - Código postal (Requerido)
  • Telefono - Teléfono de contacto (Opcional)

POST EstadoPedidosWS

Función: Consulta el estado de los pedidos. Permite consultar múltiples pedidos.

Parámetros Principales:
  • sesionwcf - Número de sesión (Requerido)
  • Pedidos - Array con los pedidos a consultar (Requerido)

POST NuevaMascotaWS

Función: Da de alta o modifica los datos de un registro de mascota.

Parámetros Principales:
  • sesionwcf - Número de sesión (Requerido)
  • ID_Cliente - ID del cliente propietario (Requerido)
  • Nombre - Nombre de la mascota (Requerido)
  • TipoAnimal - Tipo de animal (Requerido)
  • Raza - Raza del animal (Opcional)
  • FechaNacimiento - Fecha de nacimiento (Opcional)

Nota Importante

Para ver la lista completa de endpoints y sus parámetros detallados, puedes usar la pestaña "Documentación" en la página de Endpoints.

Formato JSON

Estructura JSON Válida

JSON (JavaScript Object Notation) es un formato de datos ligero y fácil de leer:

Ejemplo de JSON Válido

{
  "sesionwcf": 18,
  "Tipo": 1,
  "NIF": "12345678A",
  "Nombre": "Juan",
  "Email": "juan@ejemplo.com",
  "Direccion": "Calle Principal 123"
}

Errores Comunes

  • Comas extras: No pongas coma después del último elemento
  • Comillas simples: Usa comillas dobles ", no simples
  • Sin trailing comma: Evita la coma final en arrays y objetos
  • Llaves balanceadas: Cada { debe tener su }

Tipos de Valores en JSON

Números

Enteros o decimales sin comillas:

18, 123.45, -10

Texto

Entre comillas dobles:

"Nombre", "Juan Pérez"

Booleanos

true o false sin comillas:

true, false

Arrays

Entre corchetes, elementos separados por comas:

["item1", "item2"]

Interpretar Resultados

Tabla de Resultados

Los resultados de la API se muestran en una tabla con dos columnas:

Columna "Campo"

Muestra el nombre del campo o propiedad devuelta por la API.

Ejemplos: Id, Codigo, InfoError

Columna "Valor"

Muestra el valor devuelto por la API para ese campo.

Ejemplos: 123, "OK", {"Codigo": 0, "Mensaje": "Éxito"}

Valores JSON Anidados

Cuando el valor es un objeto JSON complejo, se formatea automáticamente para mayor legibilidad:

Ejemplo de Valor Anidado

{
  "Codigo": 0,
  "Mensaje": "Operación exitosa",
  "Data": {
    "Id": 123,
    "Nombre": "Cliente Test"
  }
}

Estados de Resultado

✅

Éxito

La operación se completó correctamente.

❌

Error

Hubo un problema con la petición. Revisa los mensajes de error.

⏳

En Proceso

La petición está siendo procesada por Verial.

Copiar Resultados

Funcionalidad de Copiar

La herramienta permite copiar los resultados de una petición al portapapeles para su posterior uso.

1

Ver Resultados

Una vez que se muestren los resultados en la tabla, verás el botón "Copiar" en la esquina superior derecha.

2

Hacer Clic

Haz clic en el botón "Copiar" para copiar toda la tabla de resultados al portapapeles.

3

Confirmación

Verás un mensaje de confirmación indicando que los datos se copiaron correctamente.

Casos de Uso

  • Documentar pruebas: Guarda los resultados para documentar tus pruebas
  • Compartir con el equipo: Copia los resultados para compartirlos con tu equipo
  • Análisis posterior: Pega los resultados en un archivo de texto o Excel para análisis
  • Depuración: Usa los resultados para identificar problemas en las peticiones

Solución de Problemas

Problemas Comunes

Error de Sintaxis JSON

Síntoma: El sistema no permite enviar la petición o muestra un error de sintaxis.

Soluciones:
  1. Verifica que todas las comillas sean dobles "
  2. Elimina comas extras al final de arrays u objetos
  3. Comprueba que las llaves {} y corchetes [] estén balanceados
  4. Usa un validador JSON online para verificar la sintaxis

Parámetros Incorrectos

Síntoma: La API devuelve un error indicando que faltan parámetros o son incorrectos.

Soluciones:
  1. Revisa la documentación del endpoint seleccionado
  2. Verifica que todos los parámetros requeridos estén presentes
  3. Comprueba que los tipos de datos sean correctos (números sin comillas, texto con comillas)
  4. Asegúrate de incluir sesionwcf si es requerido

Error de Conexión

Síntoma: No hay respuesta del servidor o timeout.

Soluciones:
  1. Verifica tu conexión a Internet
  2. Comprueba que la API de Verial esté disponible
  3. Revisa las credenciales de acceso configuradas
  4. Intenta de nuevo después de unos minutos
  5. Verifica la configuración de firewall del servidor

Sesión Expirada

Síntoma: Errores relacionados con sesión no válida o expirada.

Soluciones:
  1. Verifica que el valor de sesionwcf sea válido
  2. Asegúrate de que la sesión no haya expirado
  3. Inicia una nueva sesión en la configuración del plugin
  4. Revisa los logs del sistema para más detalles

Revisar Logs

Para obtener más información sobre errores:

  1. Navega a la página de Logs del Sistema
  2. Filtra por tipo "endpoints" o "API"
  3. Busca el timestamp de tu última prueba
  4. Revisa el mensaje de error completo
